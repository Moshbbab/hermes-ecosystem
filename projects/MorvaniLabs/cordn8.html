overview

Cordn8 is a local-first orchestration protocol designed to prevent chaos and context drift when managing multiple AI agents in a single codebase. It utilizes a "Context Spine" immutable ledger and an append-only Markdown channel to maintain a transparent, version-controlled record of all agent communications and decisions. The system includes Maya, a proactive auditor that identifies and remediates workflow failures, and supports various autonomy modes from manual approval to full autopilot. It allows users to swap between different AI models, including local Ollama or cloud-based providers, while keeping data on the local disk.

The Multi-Agent Chaos You Didn't See Coming

You've embraced AI agents. They write code, analyze data, and automate tasks. But when you scale beyond a single agent, a new problem emerges: chaos.

  • Conflicting Edits: Agents overwrite each other's work, leading to lost progress and merge conflicts.
  • Context Drift: Long-running tasks cause agents to lose their way, hallucinating or repeating past mistakes.
  • Silent Failures: Dependencies vanish, APIs change, and agents simply stop working—without telling you why.
  • Black Box Decisions: You lose visibility into why an agent did what it did, making debugging and auditing a nightmare.

This isn't just a minor inconvenience; it's an orchestration crisis that cripples productivity and trust in your AI workflows.

Cordn8 is the solution. It's not another agent. It's the local-first orchestration protocol that brings order, audibility, and proactive intelligence to your multi-agent projects. Cordn8 ensures your AI team works together seamlessly, just like a well-coordinated human team.

Experience the Cordn8 Difference

Cordn8 is engineered for teams that demand more than just a "smart autocomplete." It introduces a paradigm shift in how we manage autonomous systems:

1. Maya: Your Proactive AI Auditor

Most tools tell you that something broke. Maya tells you how to fix it—and offers to do it for you. Operating as a continuous background auditor, Maya runs 22+ validation journeys across your project. If an agent drifts from the intent, or if a deployment config is missing, Maya intercepts the workflow and provides one-click --fix remediations. It's a self-healing guardian that ensures your project never stalls.

2. Context Spine™: The Immutable Source of Truth

Long-running agent tasks fail because context degrades. Cordn8 anchors every agent to a "Context Spine"—a locked, immutable ledger of Briefs, Source Vaults, and Decisions. Agents don't guess what happened yesterday; they read the Spine. This guarantees that every AI model, from a local Ollama to a cloud-based Claude, shares the exact same ground truth, eliminating hallucination and ensuring consistent progress.

3. The Append-Only Channel: Transparent AI Collaboration

Black-box AI is a security risk. Cordn8 forces all agent communication, handoffs, and approvals through a single, human-readable, append-only Markdown file (docs/agent-channel.md). Every decision is transparent, grep-able, and checked directly into version control. Your data never leaves your disk, and your audit trail is permanent and undeniable.

4. Fluid Autonomy Modes: Control at Your Fingertips

You shouldn't have to rewrite code to change how much you trust your AI. Cordn8 provides dynamic autonomy controls:

  • MANUAL: High-security. Every agent action requires explicit human approval.
  • COPILOT: High-velocity collaboration. Routine, low-risk steps execute automatically, while high-risk changes (like filesystem writes) queue for your review.
  • AUTOPILOT: Maximum velocity. Agents execute end-to-end within their sandbox boundaries and token budgets, appending details to the channel for post-hoc audits.

Quick Start: Get Started in Minutes

Cordn8 intelligently adapts to your project, whether you're starting fresh or integrating into an existing codebase. Choose the path that fits your needs:

New Project (Greenfield)

Starting a brand new project? Cordn8 scaffolds a pristine, multi-agent ready environment from scratch.

# 1. Install Cordn8 globally
npm install -g cordn8

# 2. Initialize a new project with intelligent defaults
cordn8 init --new-project

# 3. Start the proactive watch daemon (your AI's always-on supervisor)
cordn8 watch

# 4. Dispatch your first task (Cordn8 handles the routing and coordination)
cordn8 do "Build a responsive login page with TailwindCSS"

Existing Project (Brownfield)

Integrating Cordn8 into an existing repository? Inject Cordn8's powerful coordination protocol without disrupting your current workflow.

# 1. Install Cordn8 globally
npm install -g cordn8

# 2. Initialize Cordn8 safely in your existing repository
cordn8 init --existing-project

# 3. Start the proactive watch daemon (your AI's always-on supervisor)
cordn8 watch

# 4. Dispatch a task, leveraging your existing context and codebase
cordn8 do "Refactor the user authentication module for better security"

Architecture Blueprint

Is Cordn8 Right For You?

Choose Cordn8 if:

  • You manage multiple AI agents in a single codebase and need to prevent conflicting edits, state drift, and ensure seamless collaboration.
  • You require a persistent, human-readable audit trail of every AI decision for compliance, debugging, or transparency.
  • Your security policy mandates local-first, air-gapped execution (no hosted SaaS wrappers) to keep your data private and secure.
  • You want the flexibility to seamlessly swap between AI models (Anthropic, OpenAI, local Ollama, etc.) without changing your orchestration logic.
  • You need proactive intelligence that identifies and helps fix issues before they become blockers.

Cordn8 might NOT be for you if:

  • You are running a single agent inside an IDE (like Cursor) and don't need external coordination or advanced auditing.
  • You prefer a hosted cloud platform to manage your data and don't require local-first control.
