breakdown

mem0ai / mem0

As the official memory provider for Hermes Agent, Mem0 offers a universal layer that fuses semantic, keyword, and entity matching for high-precision retrieval. It enables adaptive personalization across user, session, and agent levels through a single-pass extraction process that preserves existing data.

garrytan / gbrain

This opinionated 'brain' for Hermes Agent utilizes a self-wiring knowledge graph to automate entity and relationship extraction. It distinguishes itself by combining vector retrieval with graph-based backlink ranking, exposing over 30 MCP tools for deep integration with development environments.

vectorize-io / hindsight

Hindsight focuses on long-term learning through structured retain, recall, and reflect workflows managed via a simple two-line LLM wrapper. It is designed for enterprise-grade reliability, supporting robust storage backends like PostgreSQL and Oracle AI Database.

plastic-labs / honcho

Honcho serves as a foundational library for building stateful agents by maintaining evolving context for users and groups over time. It provides natural language querying capabilities and SDKs that format interaction history into OpenAI-compatible context.

greyhaven-ai / autocontext

This recursive context harness uses multi-role coordination and automated evaluation loops to help agents master complex tasks. It generates persistent playbooks and structured traces, allowing Hermes Agent to inspect skills and state via a CLI-first interface.

stephenschoettler / hermes-lcm

Hermes-LCM provides lossless context management using a hierarchical DAG engine that ensures conversation convergence through three-level escalation. It utilizes immutable SQLite storage and dedicated tools to search or expand compacted history without losing message integrity.

elkimek / honcho-self-hosted

This project provides a self-hosted backend for the Honcho library, enabling private cross-session memory specifically for Hermes Agent deployments. It includes automated scripts for Docker and PostgreSQL, supporting any local or OpenAI-compatible inference server.

yoloshii / ClawMem

ClawMem is a local-first context engine that eliminates cloud dependencies by performing hybrid retrieval and causal graph traversal on-device. It facilitates cross-runtime memory sharing, allowing Hermes to sync state with other agents like OpenClaw and Claude Code.

EfficientContext / ContextPilot

ContextPilot optimizes long-context inference by reordering prefixes to increase cache hits by up to 12x while significantly reducing prefill latency. It is a performance-centric choice for Hermes users running heavy workloads on backends like vLLM, SGLang, or llama.cpp.

plur-ai / plur

Plur implements a shared memory layer using an open YAML 'engram' format that allows for zero-cost local storage and retrieval. It features unique biological-inspired mechanics, including automatic memory decay and strengthening based on agent usage patterns.

AxDSan / mnemosyne

Mnemosyne is a zero-dependency memory system built on SQLite that achieves sub-millisecond latency for read and search operations. It organizes information into a three-tier BEAM architecture, separating working, episodic, and scratchpad memory for efficient processing.

keepnotes-ai / keep

Keep functions as a reflective memory system that indexes code, documents, and media with automatic OCR and summarization. It uses bidirectional edge tags to create a navigable web of relationships, accessible to agents through MCP and CLI interfaces.

amanning3390 / flowstate-qmd

This anticipatory memory system prefetches context by monitoring agent session logs to predict future information needs. It is a local-first, MCP-native solution that utilizes SQLite-vec and GGUF models to provide low-latency RAG capabilities.
