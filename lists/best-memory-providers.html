breakdown

mem0ai / mem0

The official Hermes Agent memory provider, offering a universal layer with multi-level memory for users, sessions, and autonomous states. It utilizes hybrid retrieval and temporal reasoning to rank past and future events accurately.

garrytan / gbrain

An opinionated brain for Hermes Agent featuring a self-wiring knowledge graph with zero-LLM entity extraction. It combines HNSW vector and BM25 search with an automated 'doctor' loop for autonomous memory repair.

vectorize-io / hindsight

A learning-centric memory system that implements retain, recall, and reflect workflows for long-term knowledge. It offers a simple LLM wrapper for integration and supports enterprise storage via PostgreSQL and Oracle AI Database.

plastic-labs / honcho

A stateful memory library that extracts reasoning-based conclusions from conversations rather than relying on simple vector matching. It tracks evolving entities across users and projects using hybrid BM25 and vector search.

greyhaven-ai / autocontext

A recursive context harness that uses a five-role architecture to iteratively improve agent performance through structured evaluation. It persists knowledge as reusable playbooks, datasets, and execution traces.

stephenschoettler / hermes-lcm

A lossless context management plugin utilizing a hierarchical DAG structure to prevent message loss during compaction. It pairs a local SQLite store with specialized tools for precise drill-down into conversation nodes.

AxDSan / mnemosyne

A zero-dependency memory system designed for sub-millisecond latency using local SQLite storage. Its MEMORIA engine performs structured fact extraction and recursive gap analysis for Hermes Agents.

elkimek / honcho-self-hosted

A self-hosted backend that brings Honcho's cross-session persistence to Hermes Agent privately. It supports local inference servers and OpenAI-compatible providers via a streamlined Docker, PostgreSQL, and Redis setup.

yoloshii / ClawMem

A local-first context engine that eliminates cloud dependencies and API keys for on-device memory. It integrates via MCP server and Claude Code hooks using a mix of BM25, vector search, and knowledge graphs.

plur-ai / plur

A shared memory layer for multi-agent systems that stores data as open-format YAML engrams. It employs ACT-R activation to strengthen relevant facts while decaying stale information via MCP tools.

EfficientContext / ContextPilot

A context optimization tool that accelerates inference by reordering and deduplicating shared context blocks. It boosts prefill throughput by 1.5–3× and provides native plugin support for the Hermes Agent ecosystem.

keepnotes-ai / keep

A reflective memory system featuring automated indexing for URLs, multimedia, and local directories. It combines semantic search with ranked graph traversal and edge tags to create navigable relationships.

yepyhun / Brainstack

A memory kernel that organizes temporal continuity, graph truth, and corpus retrieval into a five-shelf architecture. It provides a native Hermes-Agent MemoryProvider plugin powered by SQLite and Kuzu.

amanning3390 / flowstate-qmd

An anticipatory memory system that prefetches context by monitoring agent session logs. This MCP-native server uses SQLite-vec and GGUF models for local-first storage and retrieval.

alias8818 / hermes-tool-slimmer

A utility that reduces tool-schema token overhead through BM25 ranking and keyword selection. It supports native Hermes tools and MCP integrations, featuring a diagnostics dashboard for tracking token savings.
